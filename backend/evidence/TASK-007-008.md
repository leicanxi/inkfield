# TASK-007 / TASK-008 验证记录

验证日期：2026-07-16（Asia/Shanghai）

## 已实现

- `user_week_capacities`、`user_week_allocation_sets`、`user_week_allocations`、`user_week_runs` ORM 与第四段 Alembic 迁移。
- active AllocationSet 部分唯一索引、递增 allocation revision、已应用 set/item 定义冻结触发器和用户周 rollover 幂等唯一约束。
- 容量分配策略：先保护 `max(planned, actual)`，再按截止风险、优先级满足最低预算和期望预算；paused 排除，容量不足的 active 项目保留为 `unfunded/0/capacity_shortage`。
- 同用户周重分配采用 Capacity → 旧 AllocationSet → 旧 Allocation items → Projects → WeekPlans 的稳定锁序，追加新 revision 并重绑可变周计划。
- `week_plans`、`tasks`、`task_dependencies`、`task_events` ORM 与第五段 Alembic 迁移。
- 每项目/周至多一个 current WeekPlan、同租户复合外键、Task due date 周内约束、TaskEvent UPDATE/DELETE 拒绝触发器。
- TaskEvent 应用服务按 expected task version 更新 Task、Project 事件水位及 allocation/capacity actual 汇总；耗时使用累计覆盖值。
- 完成、重开、跳过、恢复、延期、取消、替换、累计耗时状态规则；重开和延期保留历史投入及首次完成事实。
- 硬依赖 DAG 校验、查询时派生 `is_blocking`、`is_blocked` 和未满足前置任务。
- UserWeek、Project WeekPlan、Task 查询及 TaskEvent 写入的既定 `/api/v1` 路由；事件写入同时使用 API 与领域事件幂等键。
